## Commands

```bash
# Development
npm run dev              # tsx watch (hot reload)
npm run start:dev        # tsx (single run)
npm run build            # tsc ‚Üí dist/

# Migrations
npm run migrate:create   # interactive prompt ‚Üí creates timestamped migration file
npm run migrate          # applies all pending migrations to rqlite

# Infrastructure (run from repo root, not inside docker/)
docker compose -f docker/docker-compose.yml --env-file .env up -d --build
docker compose -f docker/docker-compose.yml logs -f rqlite
docker compose -f docker/docker-compose.yml down -v   # destroys persisted data

# Validate rqlite leader is up
curl -s http://localhost:8000/status
```

There is no test suite. There is no lint script.

## Environment

Copy `.env.example` to `.env`. Key variables:
- `RQLITE_ADVERTISE_IP` ‚Äî leave blank for single-machine dev; set to the machine's routable IP for multi-node clusters
- `NODE_ID` ‚Äî unique integer per node (start with `1` for the leader)
- `JOIN_NODE` ‚Äî blank for the leader; `<LEADER_IP>:4002` for workers (port 4002 is Raft)
- `PORT` ‚Äî HTTP server port (defaults to `3000`)
- `JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` ‚Äî fallback to insecure dev strings if unset

## Architecture

The server is an **Express 5 + TypeScript ESM** app that talks to **rqlite** (SQLite with Raft consensus) over HTTP. There is no ORM ‚Äî SQL is generated by the `core/db` layer and sent to rqlite's REST API.

### Request path

```
HTTP request
  ‚Üí HttpServer (Express wrapper, src/core/http/http-server.ts)
  ‚Üí BaseRouter<T> subclass (src/routes/)       ‚Äî maps HTTP verbs to controller calls
  ‚Üí Controller (src/controllers/)              ‚Äî validates request, calls service
  ‚Üí Service (src/services/)                   ‚Äî business logic, calls model methods
  ‚Üí Model<T> (src/core/db/model.ts)           ‚Äî CRUD via SQLBuilder
  ‚Üí SQLBuilder<T> (src/core/db/sql-builder.ts)‚Äî builds SQL string via squel
  ‚Üí rqlite() / sql() (src/core/db/shared.ts)  ‚Äî POSTs to rqlite HTTP API
```

**Hard rule**: SQL must never be written outside `src/core/db/`. Services and models use the abstractions provided there.

### Key abstractions

| File | Role |
|------|------|
| `src/core/db/shared.ts` | `rqlite(queries, endpoint)` ‚Äî raw client; `sql(query)` ‚Äî convenience wrapper for single queries. Endpoints: `query` (SELECT), `execute` (write), `request` (either). |
| `src/core/db/sql-builder.ts` | `SQLBuilder<T>` ‚Äî builds INSERT/SELECT/UPDATE/DELETE strings via `squel`. Stateful per call; call `create/read/update/delete` then `.raw()`. |
| `src/core/db/model.ts` | `Model<T>` ‚Äî generic CRUD class. Domain models instantiate this. Auto-assigns ULID on `create()`. |
| `src/core/db/migrator.ts` | `Migrator.build()` ‚Äî creates `_migrations` table, loads files from `src/core/db/migrations/`, runs pending ones in filename order. |
| `src/core/http/http-server.ts` | `HttpServer` ‚Äî wraps Express, mounts CORS + JSON + Swagger, then caller's routes. Add global middleware via `.use()` before `.start()`. |
| `src/routes/base-router.ts` | `BaseRouter<T>` ‚Äî abstract class wiring `GET /`, `GET /:id`, `POST /`, `PUT /:id`, `DELETE /:id` to a controller. Pass per-operation middlewares in the constructor (not as a class field ‚Äî see file comment). |
| `src/core/auth/jwt-service.ts` | Singleton `JwtService` ‚Äî issues/verifies access (15 m) and refresh (7 d) tokens. |
| `src/core/auth/middleware.ts` | Express middleware that validates the `Authorization: Bearer` header and attaches the decoded payload to `req`. |

```
@models/*  ‚Üí src/models/

```

### Adding a new migration

```bash
npm run migrate:create   # prompts for a description, writes a timestamped file
# ‚Üí edit the generated file in src/core/db/migrations/
npm run migrate          # applies it
```

Migration files export a `migration: Migration` object with `id`, `up: string[]`, and optional `down: string[]`. The `id` must match the filename stem exactly. Migrations run in filename (timestamp) order.

### Layer responsibilities

| Layer | Allowed | Forbidden |
|-------|---------|-----------|
| `src/core/db/` | SQL generation, rqlite connection, migrations | Business logic |
| `src/models/` | Instantiate `Model<T>`, expose domain-shaped methods | Raw SQL, business rules |
| `src/services/` | Business logic, orchestrate model calls | Raw SQL, direct rqlite calls |
| `src/controllers/` | Parse request, call service, return response | Business logic, SQL |
| `src/utils/` | Generic, domain-agnostic helpers | Domain-specific logic |
| `src/types/` | Global TS interfaces and type definitions | Runtime code |

### Domain entities

- **workspaces** ‚Äî user working areas; the workspace `id` doubles as the root page identifier
- **users** ‚Äî system users; many-to-many relationship with workspaces
- **pages** ‚Äî content units owned by a user; can be structured hierarchically via `page_hubs`
- **page_hubs** ‚Äî joins that connect a root page to child pages (tree/graph structure)
- **page_columns** ‚Äî typed column definitions per root page (`text`, `numeric`, `select`, `date`, `checkbox`)
- **page_columns_values** ‚Äî stores the value of a column for a specific page (EAV-style)

### Conventions

- Filenames: kebab-case (`users-service.ts`, `sql-builder.ts`)
- IDs: ULID (generated by `model.ts` on create; 26 chars, TEXT column)
- The `squel` library does not escape values ‚Äî string interpolation in `sql-builder.ts` uses single-quote wrapping. Keep SQL confined to that layer.
- Swagger docs live at `/api-docs` (auto-mounted by `HttpServer`).
